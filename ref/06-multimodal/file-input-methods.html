File Input Methods for the Gemini API

This page explains how to include media files (images, audio, video, documents) in Gemini API requests.

Input Method Comparison

MethodBest ForMax SizePersistence
Inline DataQuick testing, small files, real-time100 MB (50 MB PDFs)None — sent per request
File API UploadLarge files, multiple uses2 GB per file, 20 GB project48 hours
File API GCS URILarge GCS files, multiple uses2 GB per file, unlimited storageNone — fetched per request; 30-day access
External URLsPublic/cloud data without re-upload100 MB per requestNone — fetched per request

1. Inline Data

Best for quick testing and small files. Data sent directly in request payload with no persistence between requests.

Python

from google import genai
from google.genai import types
import pathlib

client = genai.Client()

filepath = pathlib.Path('my_local_file.pdf')
response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[
        types.Part.from_bytes(
            data=filepath.read_bytes(),
            mime_type='application/pdf',
        ),
        "Summarize this document"
    ]
)
print(response.text)

JavaScript

import { GoogleGenAI } from "@google/genai";
import * as fs from "fs";
import * as path from "path";

const ai = new GoogleGenAI({});

const filePath = path.join('content', 'my_local_file.pdf');
const contents = [
    { text: "Summarize this document" },
    {
        inlineData: {
            mimeType: 'application/pdf',
            data: fs.readFileSync(filePath).toString("base64")
        }
    }
];

const response = await ai.models.generateContent({
    model: "gemini-3.5-flash",
    contents: contents
});
console.log(response.text);

REST

B64_CONTENT=$(base64 -w 0 my_local_file.pdf)
curl -X POST "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -H 'Content-Type: application/json' \
  -d '{
    "contents": [{
      "parts": [
        {"text": "Summarize this document"},
        {"inlineData": {"mimeType": "application/pdf", "data": "'"${B64_CONTENT}"'"}}
      ]
    }]
  }'

2. Gemini File API Upload

Ideal for larger files or frequent reuse. Files persist for 48 hours after upload. Supports resumable uploads.

Python

from google import genai

client = genai.Client()

audio_file = client.files.upload(file="path/to/your/sample.mp3")

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=["Describe this audio clip", audio_file]
)
print(response.text)

JavaScript

import { GoogleGenAI, createUserContent, createPartFromUri } from "@google/genai";

const ai = new GoogleGenAI({});

const myfile = await ai.files.upload({
    file: "path/to/your/sample.mp3",
    config: { mimeType: "audio/mpeg" },
});

const response = await ai.models.generateContent({
    model: "gemini-3.5-flash",
    contents: createUserContent([
        "Describe this audio clip",
        createPartFromUri(myfile.uri, myfile.mimeType),
    ]),
});
console.log(response.text);

REST (Resumable Upload)

AUDIO_PATH="path/to/sample.mp3"
MIME_TYPE=$(file -b --mime-type "${AUDIO_PATH}")
NUM_BYTES=$(wc -c < "${AUDIO_PATH}")

curl "${BASE_URL}/upload/v1beta/files" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -D "upload-header.tmp" \
  -H "X-Goog-Upload-Protocol: resumable" \
  -H "X-Goog-Upload-Command: start" \
  -H "X-Goog-Upload-Header-Content-Length: ${NUM_BYTES}" \
  -H "X-Goog-Upload-Header-Content-Type: ${MIME_TYPE}" \
  -H "Content-Type: application/json" \
  -d "{'file': {'display_name': 'AUDIO'}}"

3. File API GCS URI Registration

For files already in Google Cloud Storage. Registration valid up to 30 days. Requires IAM setup.

Setup Steps

  1. Enable the Gemini API in your Google Cloud project
  2. Create Service Agent: 
    gcloud beta services identity create \
  --service=generativelanguage.googleapis.com \
  --project=<your_project>
  3. Grant "Storage Object Viewer" IAM role to the service agent on your specific buckets

Python (Authentication outside Google Cloud)

from google.oauth2.service_account import Credentials
from google import genai
from google.genai.types import Part

GCS_READ_SCOPES = [
    'https://www.googleapis.com/auth/devstorage.read_only',
    'https://www.googleapis.com/auth/cloud-platform'
]

credentials = Credentials.from_service_account_file(
    'service-account.json',
    scopes=GCS_READ_SCOPES
)

client = genai.Client()

registered_gcs_files = client.files.register_files(
    uris=["gs://my_bucket/some_object.pdf", "gs://bucket2/object2.txt"],
    auth=credentials
)

for f in registered_gcs_files.files:
    response = client.models.generate_content(
        model="gemini-3.5-flash",
        contents=[
            Part.from_uri(file_uri=f.uri, mime_type=f.mime_type),
            "Summarize this file."
        ],
    )
    print(response.text)

Python (Authentication on Google Cloud)

import google.auth

GCS_READ_SCOPES = [
    'https://www.googleapis.com/auth/devstorage.read_only',
    'https://www.googleapis.com/auth/cloud-platform'
]

credentials, project = google.auth.default(scopes=GCS_READ_SCOPES)

REST (CLI)

access_token=$(gcloud auth application-default print-access-token)
project_id=$(gcloud config get-value project)

curl -X POST https://generativelanguage.googleapis.com/v1beta/files:register \
    -H 'Content-Type: application/json' \
    -H "Authorization: Bearer ${access_token}" \
    -H "x-goog-user-project: ${project_id}" \
    -d '{"uris": ["gs://bucket/object1", "gs://bucket/object2"]}'

4. External HTTP / Signed URLs

For publicly accessible HTTPS or pre-signed URLs. No re-uploading needed. Note: Gemini 2.0 models are not supported.

Python

from google import genai
from google.genai.types import Part

uri = "https://ontheline.trincoll.edu/images/bookdown/sample-local-pdf.pdf"

client = genai.Client()
response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[
        Part.from_uri(
            file_uri=uri,
            mime_type="application/pdf",
        ),
        "Summarize this file"
    ],
)
print(response.text)

JavaScript

import { GoogleGenAI, createPartFromUri } from '@google/genai';

const client = new GoogleGenAI({});
const uri = "https://ontheline.trincoll.edu/images/bookdown/sample-local-pdf.pdf";

const response = await client.models.generateContent({
    model: 'gemini-3.5-flash',
    contents: [
        createPartFromUri(uri, "application/pdf"),
        "summarize this file",
    ],
});
console.log(response.text);

REST

curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent" \
  -H 'x-goog-api-key: $GEMINI_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "contents":[{
      "parts":[
        {"text": "Summarize this pdf"},
        {"file_data": {
          "mime_type":"application/pdf",
          "file_uri": "https://ontheline.trincoll.edu/images/bookdown/sample-local-pdf.pdf"
        }}
      ]
    }]
  }'

Supported MIME Types

Text

text/html, text/css, text/plain, text/xml, text/csv, text/rtf, text/javascript

Application

application/json, application/pdf

Image

image/bmp, image/jpeg, image/png, image/webp

Video

video/mp4, video/mpeg, video/quicktime, video/avi, video/x-flv, video/mpg, video/webm, video/wmv, video/3gpp

Audio (via File API)

audio/mpeg and other formats supported through File API

Key Parameters

  • file_uri: URL or GCS path to the file
  • mime_type: Content type of the file (required for proper processing)
  • data: Base64-encoded file content (for inline data)
  • inlineData: JavaScript object containing MIME type and base64 data
  • file_data: REST parameter for file references
  • uris: Array of GCS paths for batch registration
  • auth: Authentication credentials for GCS operations

Best Practices

  • Use inline data for small, transient files
  • Use the File API for larger or frequently used files
  • Use external URLs for data already hosted online
  • Always specify correct MIME types for proper file handling
  • Implement error handling for network failures and API issues
  • For GCS, grant the Gemini API Service Agent only the needed Storage Object Viewer role on specific buckets
  • Ensure signed URLs have appropriate expiration and limited permissions

Limitations

  • File sizes and types vary by method
  • Inline data increases request payload size
  • File API uploads expire after 48 hours
  • External URL fetching limited to 100 MB with specific content types
  • External URLs not supported on Gemini 2.0 models
  • GCS registration requires proper IAM setup and OAuth token management
