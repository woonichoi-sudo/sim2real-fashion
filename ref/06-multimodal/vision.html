Image Understanding with Gemini API

Google's Gemini models support multimodal image processing for tasks including image captioning, classification, and visual question answering without having to train specialized ML models.

Passing Images to Gemini

Method 1: Inline Image Data

Suitable for smaller files (under 20 MB total request size). Supports Base64 encoding or direct local file reading.

Python — Local File

from google import genai
import pathlib

client = genai.Client()

image_path = pathlib.Path("path/to/image.jpg")

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[
        {"mime_type": "image/jpeg", "data": image_path.read_bytes()},
        "Describe what you see in this image"
    ]
)
print(response.text)

Python — PIL Image

from PIL import Image
from google import genai

client = genai.Client()
image = Image.open("/path/to/organ.png")

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[image, "Tell me about this instrument"]
)
print(response.text)

Python — Image from URL (Base64)

import requests
import base64
from google import genai
from google.genai import types

client = genai.Client()

image_url = "https://example.com/image.jpg"
response = requests.get(image_url)
image_data = base64.b64encode(response.content).decode("utf-8")

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[
        types.Part.from_bytes(
            data=base64.b64decode(image_data),
            mime_type="image/jpeg"
        ),
        "Describe this image"
    ]
)
print(response.text)

JavaScript

import { GoogleGenAI } from "@google/genai";
import * as fs from "fs";

const ai = new GoogleGenAI({});

const imageData = fs.readFileSync("path/to/image.jpg");
const base64Image = imageData.toString("base64");

const response = await ai.models.generateContent({
    model: "gemini-3.5-flash",
    contents: [
        {
            inlineData: {
                mimeType: "image/jpeg",
                data: base64Image
            }
        },
        { text: "Describe what you see in this image" }
    ]
});
console.log(response.text);

REST

B64_IMAGE=$(base64 -w 0 path/to/image.jpg)
curl -X POST "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [{
      "parts": [
        {"inlineData": {"mimeType": "image/jpeg", "data": "'"${B64_IMAGE}"'"}},
        {"text": "Describe this image"}
      ]
    }]
  }'

Method 2: File API Upload

Recommended for larger files or repeated use of the same image across multiple requests.

from google import genai

client = genai.Client()

uploaded_image = client.files.upload(file="path/to/large_image.jpg")

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[uploaded_image, "Analyze this image in detail"]
)
print(response.text)

Prompting with Multiple Images

from google import genai

client = genai.Client()

image1 = open("image1.jpg", "rb").read()
image2 = open("image2.jpg", "rb").read()

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[
        {"mime_type": "image/jpeg", "data": image1},
        {"mime_type": "image/jpeg", "data": image2},
        "Compare these two images"
    ]
)
print(response.text)

Object Detection

The model can detect prominent items and return bounding box coordinates normalized to a 0–1000 scale in [ymin, xmin, ymax, xmax] format. Coordinates require descaling based on original image dimensions.

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=[
        image,
        "Detect all objects. Return bounding boxes as JSON with format: "
        "[{\"label\": \"name\", \"box\": [ymin, xmin, ymax, xmax]}]"
    ]
)

# Descaling example: if image is 1920x1080
# actual_ymin = (ymin / 1000) * 1080
# actual_xmin = (xmin / 1000) * 1920

Custom instructions are supported: detecting specific colored objects, applying custom labels, filtering by object type.

  • Scene understanding
  • Chart and diagram analysis
  • Enhanced accuracy for specific use cases through additional training

Limitations and Key Technical Information

Token Calculation

  • Images with both dimensions ≤384 pixels: 258 tokens
  • Larger images: divided into 768x768 pixel tiles at 258 tokens each
  • Formula: divide each dimension by crop unit size (roughly floor(min(width, height) / 1.5))
  • Example: a 960×540 image = 6 tiles = 1,548 tokens

Media Resolution Parameter

The media_resolution parameter controls "the maximum number of tokens allocated per input image or video frame," balancing detail recognition against token usage and latency.

Limits

  • Maximum 3,600 image files per request
  • 20 MB inline request size limit

Tips and Best Practices

  • Verify correct image rotation before sending
  • Use clear, non-blurry images for best results
  • Place text prompts after image parts in the contents array
  • For very high-resolution images, consider whether full resolution is needed (use media_resolution to control token cost)
  • For repeated analysis of the same image, upload once via File API and reuse the URI
