Files API Documentation

This page documents Google's Files API for the Gemini API, explaining how to upload and manage media files for use with Gemini models.

Key Operations

1. Upload a File

Use the Files API when total request size exceeds 100 MB, for PDFs over 50 MB, or for files you reuse across multiple requests.

Python

from google import genai

client = genai.Client()

myfile = client.files.upload(file="path/to/sample.mp3")

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=["Describe this audio clip", myfile]
)
print(response.text)

JavaScript

import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({});

const myfile = await ai.files.upload({
    file: "path/to/sample.mp3",
    config: { mimeType: "audio/mpeg" }
});

const response = await ai.models.generateContent({
    model: "gemini-3.5-flash",
    contents: ["Describe this audio clip", { fileData: { fileUri: myfile.uri, mimeType: myfile.mimeType } }]
});
console.log(response.text);

REST (Resumable Upload)

AUDIO_PATH="path/to/sample.mp3"
MIME_TYPE=$(file -b --mime-type "${AUDIO_PATH}")
NUM_BYTES=$(wc -c < "${AUDIO_PATH}")
BASE_URL="https://generativelanguage.googleapis.com"

# Step 1: Start upload
curl "${BASE_URL}/upload/v1beta/files" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -D "upload-header.tmp" \
  -H "X-Goog-Upload-Protocol: resumable" \
  -H "X-Goog-Upload-Command: start" \
  -H "X-Goog-Upload-Header-Content-Length: ${NUM_BYTES}" \
  -H "X-Goog-Upload-Header-Content-Type: ${MIME_TYPE}" \
  -H "Content-Type: application/json" \
  -d "{'file': {'display_name': 'AUDIO'}}"

# Get upload URL from response header
upload_url=$(grep -i "x-goog-upload-url: " upload-header.tmp | cut -d" " -f2 | tr -d "\r")

# Step 2: Upload the file bytes
curl "${upload_url}" \
  -H "X-Goog-Upload-Command: upload, finalize" \
  -H "X-Goog-Upload-Offset: 0" \
  --data-binary "@${AUDIO_PATH}" > upload-response.tmp

2. Get Metadata for a File

Verify that files were successfully stored and retrieve file information using files.get.

# Python
file_info = client.files.get(name=myfile.name)
print(file_info.state)  # PROCESSING, ACTIVE, or FAILED

# JavaScript
const fileInfo = await ai.files.get({ name: myfile.name });

# REST
curl "https://generativelanguage.googleapis.com/v1beta/${FILE_NAME}" \
  -H "x-goog-api-key: $GEMINI_API_KEY"

3. List Uploaded Files

# Python
for file in client.files.list():
    print(file.name, file.display_name, file.state)

# REST
curl "https://generativelanguage.googleapis.com/v1beta/files" \
  -H "x-goog-api-key: $GEMINI_API_KEY"

4. Delete Uploaded Files

# Python
client.files.delete(name=myfile.name)

# REST
curl -X DELETE "https://generativelanguage.googleapis.com/v1beta/${FILE_NAME}" \
  -H "x-goog-api-key: $GEMINI_API_KEY"

Usage Info / Storage Limits

  • Up to 20 GB of files per project
  • Per-file maximum size: 2 GB
  • PDF files: 50 MB limit for inline; use Files API for larger
  • Automatic deletion after 48 hours (manual deletion available)
  • Available at no cost across all regions

Supported Media Types

  • Audio: MP3, WAV, and other formats
  • Images: JPEG, PNG, WEBP, HEIC, HEIF
  • Videos: MP4, AVI, MOV, and other formats
  • Documents: PDF, plain text

File Prompting Strategies (Prompt Design Best Practices)

  • Be specific with instructions to avoid generic responses
  • Use few-shot examples to demonstrate desired output patterns
  • Break complex tasks into manageable steps
  • Specify output format (JSON, markdown, HTML, etc.)
  • Place single images before text for potentially better performance
  • Keep temperature at default 1.0 for Gemini 3 models

Troubleshooting Your Multimodal Prompt

When prompts underperform:

  • Add hints about which image sections to focus on
  • Ask the model to describe images before performing tasks
  • Test with lower temperature settings to reduce hallucination
  • Explicitly request step-by-step reasoning for complex problems
