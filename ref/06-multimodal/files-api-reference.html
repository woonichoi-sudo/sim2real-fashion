Files API Reference

The Gemini API Files reference covers file handling capabilities, allowing developers to upload and reuse media files across multiple requests.

REST Resource: files

Resource: File

A file uploaded to the API. Files are stored separately from prompts, enabling reuse through their unique identifiers.

Methods

files.get

Retrieves metadata for a specific file by name.

HTTP MethodGET
Endpointhttps://generativelanguage.googleapis.com/v1beta/{name=files/*}

Path parameter: name (string, required) — File identifier

# Python
file_info = client.files.get(name="files/abc123")

# REST
curl "https://generativelanguage.googleapis.com/v1beta/files/abc123" \
  -H "x-goog-api-key: $GEMINI_API_KEY"

files.list

Lists all files owned by the requesting project with pagination support.

HTTP MethodGET
Endpointhttps://generativelanguage.googleapis.com/v1beta/files

Query parameters:

  • pageSize (integer, optional): Max files per page, default 10, max 100
  • pageToken (string, optional): Pagination token for subsequent pages

Response fields:

  • files[]: Array of File objects
  • nextPageToken: Pagination token for subsequent calls
# Python
for file in client.files.list():
    print(file.name, file.display_name, file.state)

# REST
curl "https://generativelanguage.googleapis.com/v1beta/files?pageSize=20" \
  -H "x-goog-api-key: $GEMINI_API_KEY"

files.delete

Removes a file permanently.

HTTP MethodDELETE
Endpointhttps://generativelanguage.googleapis.com/v1beta/{name=files/*}

Path parameter: name (string, required) — File identifier

# Python
client.files.delete(name="files/abc123")

# REST
curl -X DELETE "https://generativelanguage.googleapis.com/v1beta/files/abc123" \
  -H "x-goog-api-key: $GEMINI_API_KEY"

files.register

Registers Google Cloud Storage URIs as files without copying them.

HTTP MethodPOST
Endpointhttps://generativelanguage.googleapis.com/v1beta/files:register

Request body fields:

  • uris[] (string array, required): The Google Cloud Storage URIs to register
# REST
curl -X POST https://generativelanguage.googleapis.com/v1beta/files:register \
    -H 'Content-Type: application/json' \
    -H "Authorization: Bearer ${access_token}" \
    -H "x-goog-user-project: ${project_id}" \
    -d '{"uris": ["gs://bucket/object1", "gs://bucket/object2"]}'

media.upload

Creates and uploads new files through resumable upload protocol.

HTTP Method (media)POST https://generativelanguage.googleapis.com/upload/v1beta/files
HTTP Method (metadata)POST https://generativelanguage.googleapis.com/v1beta/files

Video Processing Note

Video processing requires polling until the file state reaches ACTIVE before using in content generation:

import time
from google import genai

client = genai.Client()

video_file = client.files.upload(file="path/to/video.mp4")

# Poll until ACTIVE
while video_file.state == "PROCESSING":
    time.sleep(5)
    video_file = client.files.get(name=video_file.name)

if video_file.state == "FAILED":
    raise ValueError("Video processing failed")

# Now safe to use
response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents=["Describe this video", video_file]
)
print(response.text)
