Gemini API: Generating Content - REST Reference

Note: URL https://ai.google.dev/api/rest/v1/SafetySetting redirects to the general "Generating Content" reference page which includes SafetySetting as part of the request body.

Core Methods

1. models.generateContent

Endpoint: POST https://generativelanguage.googleapis.com/v1beta/{model=models/*}:generateContent

Path Parameter:

  • model (string, required): Format models/{model}

2. models.streamGenerateContent

Endpoint: POST https://generativelanguage.googleapis.com/v1beta/{model=models/*}:streamGenerateContent

Returns streamed responses as GenerateContentResponse instances.

Supported Content Types

  • Text: Direct string input
  • Images: JPEG, PNG (uploaded via Files API)
  • Audio: MP3, WAV (requires upload via Files API)
  • Video: MP4 (requires processing - polling until ACTIVE state)
  • PDF: Documents (uploaded via Files API)
  • Code: Execution and analysis

Key Features

  • JSON Mode: Returns responses in JSON format with optional schema validation using response_mime_type: "application/json" and responseSchema
  • Function Calling: Enables tool invocation through tools[] parameter with FunctionDeclaration objects
  • Code Execution: Model can write and execute code; response includes both executable code parts and execution results
  • Caching: Reference pre-computed content via cachedContent parameter to reduce token costs
  • System Instructions: Set custom behavior through systemInstruction field (text-only currently)

SDK Support

  • Python (google-genai)
  • Node.js/TypeScript
  • Go
  • Java
  • C#

Authorization

Requires API key passed as key query parameter or in request headers.
