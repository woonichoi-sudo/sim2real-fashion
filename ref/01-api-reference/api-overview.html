Gemini API Reference

Primary Endpoints

The API offers several approaches for interacting with Gemini models:

  • Interactions API (CreateInteraction): "Optimized for agentic workflows, server-side state management, and complex multi-modal, multi-turn conversations" (v1beta only)
  • generateContent: Standard REST endpoint. Returns full response in a single package.
  • streamGenerateContent: Uses Server-Sent Events (SSE) for progressive response delivery.
  • BidiGenerateContent: WebSocket-based bi-directional streaming (Live API) for real-time conversations.
  • batchGenerateContent: Submits multiple generateContent requests in batch mode.
  • embedContent: Generates text embedding vectors.
  • Gen Media APIs: Specialized endpoints for image and video generation.
  • Platform APIs: File upload, token counting, etc.

Authentication

All requests require an API key. Pass it via the x-goog-api-key header or as a query parameter ?key=$GEMINI_API_KEY.

API keys can be created in Google AI Studio.

-H "x-goog-api-key: $GEMINI_API_KEY"
-H 'Content-Type: application/json'

Content Generation

Request Body Structure

FieldTypeDescription
contentsContent[] (required)Array of Content objects representing conversation turns
rolestring"user" or "model"
partsPart[]Array of Part objects containing actual data (text, images, etc.)
textstringText content
inline_data (Blob)objectFor embedding raw media directly in requests; contains mime_type and data (base64)

Response Body Structure

FieldTypeDescription
candidatesCandidate[]Array of Candidate objects
finishReasonstringReason generation stopped (e.g., STOP)
indexintegerResponse candidate index
usageMetadataobjectToken usage information
modelVersionstringModel version used
responseIdstringResponse identifier

Request Examples

Text-only Prompt

POST https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=$GEMINI_API_KEY
Content-Type: application/json

{
  "contents": [{
    "parts": [{"text": "Explain how AI works in a single paragraph."}]
  }]
}

Multimodal Prompt (Text and Image)

{
  "contents": [{
    "parts": [
      {"inline_data": {"mime_type": "image/jpeg", "data": "[base64_encoded_image]"}},
      {"text": "What is in this picture?"}
    ]
  }]
}

Multi-turn Conversation (Chat)

{
  "contents": [
    {"role": "user", "parts": [{"text": "Hello."}]},
    {"role": "model", "parts": [{"text": "Hello! How can I help?"}]},
    {"role": "user", "parts": [{"text": "Write a poem about the ocean."}]}
  ]
}

Standard Response Example

{
  "candidates": [{
    "content": {
      "parts": [{"text": "At its core, Artificial Intelligence..."}],
      "role": "model"
    },
    "finishReason": "STOP",
    "index": 1
  }],
  "usageMetadata": {
    "promptTokenCount": 10,
    "candidatesTokenCount": 50,
    "totalTokenCount": 60
  },
  "modelVersion": "gemini-2.0-flash"
}

Live API (BidiGenerateContent)

WebSocket-based bi-directional streaming for real-time conversations. Used for audio and low-latency interactive applications.

Key Takeaways

  • Conversation history maintained by including multiple Content objects with alternating "user" and "model" roles.
  • SDKs default to v1beta; use v1 for stable production environments.
  • The Interactions API is optimized for agentic workflows and is v1beta only.
