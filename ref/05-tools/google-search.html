Grounding with Google Search

This documentation page describes how to use Google Search grounding with the Gemini API to enhance response accuracy and provide source citations.

Key Capabilities

  • Increase factual accuracy: Reduce model hallucinations by basing responses on real-world information
  • Access current information beyond the model's training data
  • Generate citations linking claims to verifiable sources

How Grounding with Google Search Works

When enabled, the google_search tool automates a five-step workflow:

  1. User submits a prompt with the tool enabled
  2. Model analyzes whether search would improve the answer
  3. Model executes one or more search queries automatically
  4. Search results are processed and synthesized
  5. Response returned with grounding metadata including queries, results, and citations

Code Examples

Python

from google import genai
from google.genai import types

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents="What's the current price of Google stock?",
    config=types.GenerateContentConfig(
        tools=[types.Tool(google_search=types.GoogleSearch())]
    )
)
print(response.text)

Understanding the Grounding Response

Successful grounded responses include groundingMetadata containing:

  • webSearchQueries: Array of search terms used
  • searchEntryPoint: HTML/CSS for search widget rendering
  • groundingChunks: Web sources with URI and title
  • groundingSupports: Links between text segments and source indices
# Accessing grounding metadata (Python)
metadata = response.candidates[0].grounding_metadata
print(metadata.web_search_queries)
for chunk in metadata.grounding_chunks:
    print(chunk.web.uri, chunk.web.title)

Attributing Sources with Inline Citations

Creating inline citations from metadata:

  1. Extract supports and chunks from groundingMetadata
  2. Sort supports by end_index (descending)
  3. Create markdown-style links: [index](uri)
  4. Insert citations at segment endpoints in the response text
def add_citations(response):
    text = response.candidates[0].content.parts[0].text
    supports = response.candidates[0].grounding_metadata.grounding_supports
    chunks = response.candidates[0].grounding_metadata.grounding_chunks

    # Sort supports by end_index descending to insert from end
    sorted_supports = sorted(supports, key=lambda s: s.segment.end_index, reverse=True)

    for support in sorted_supports:
        end = support.segment.end_index
        citations = "".join(
            f"[{i+1}]({chunks[i].web.uri})"
            for i in support.grounding_chunk_indices
        )
        text = text[:end] + citations + text[end:]
    return text

Pricing

  • For Gemini 3 models: billed for each search query that the model decides to execute. Multiple queries within a single request count separately; empty queries are ignored.
  • For Gemini 2.5 and older models: billed per prompt (not per query)
