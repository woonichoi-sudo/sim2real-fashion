Grounding with Google Maps

Overview

The Grounding with Google Maps feature for the Gemini API integrates location-aware functionality into applications using current Google Maps data, including over 250 million places.

Key Features

  • Accurate, location-aware responses leveraging Google Maps' extensive database
  • Enhanced personalization based on user-provided location coordinates
  • Contextual widgets for interactive map displays alongside generated content

Why and When to Use Grounding with Google Maps

  • Finding nearby restaurants, cafes, and services
  • Location-based recommendations
  • Trip planning and itinerary assistance
  • Geographic-specific queries requiring current factual data

API Methods and Parameters

Python

from google import genai
from google.genai import types

client = genai.Client()

response = client.models.generate_content(
    model='gemini-3.5-flash',
    contents="Find me a good Italian restaurant nearby",
    config=types.GenerateContentConfig(
        tools=[types.Tool(google_maps=types.GoogleMaps())],
        tool_config=types.ToolConfig(
            retrieval_config=types.RetrievalConfig(
                lat_lng=types.LatLng(latitude=34.050481, longitude=-118.248526)
            )
        )
    )
)
print(response.text)

JavaScript

import { GoogleGenAI } from "@google/genai";

const ai = new GoogleGenAI({});

const response = await ai.models.generateContent({
    model: "gemini-3.5-flash",
    contents: "Find me a good Italian restaurant nearby",
    config: {
        tools: [{ googleMaps: {} }],
        toolConfig: {
            retrievalConfig: {
                latLng: { latitude: 34.050481, longitude: -118.248526 }
            }
        }
    }
});
console.log(response.text);

Key Parameters

  • googleMaps: Tool configuration object to enable the feature
  • enableWidget: Boolean to control whether a contextual map widget token is returned
  • latLng: Location coordinates (latitude/longitude) for location context
  • retrievalConfig: Configuration for location-based retrieval

Understanding the Grounding Response

Responses include grounding metadata:

  • groundingChunks: Array of Maps sources with uri, title, and placeId
  • groundingSupports: Text span connections to sources
  • googleMapsWidgetContextToken: Token for rendering the contextual widget (when enableWidget: true)
  • webSearchQueries: Associated search queries used

Display the Google Maps Contextual Widget

Enable the widget by setting enableWidget: true. Render using the custom HTML element:

<gmp-place-contextual context-token="YOUR_TOKEN_HERE"></gmp-place-contextual>

Service Usage Requirements (Attribution)

When displaying responses:

  • Google Maps sources must immediately follow the generated content
  • Sources must be viewable within one user interaction
  • Attribution text must read "Google Maps"
  • Text styling: Roboto font, 12–16sp size, 4.5:1 contrast ratio for accessibility
  • Display source titles and links
  • Inform users that Google Maps data supports the answers

Pricing and Rate Limits

  • Cost: $25 per 1,000 grounded prompts
  • Free tier: 500 requests per day

Limitations

  • Text-based responses only (contextual widget is separate from model output)
  • Tool disabled by default; must be explicitly enabled in the tools parameter
  • Restricted use in certain high-risk activities and prohibited territories per Google Maps Platform terms
