How Tool Execution Works

Built-in Tool Flow

Google-managed tools execute within a single API call. The model receives your prompt, runs necessary tools on Google's servers, and returns the final answer. No additional round-trips required from your application.

Custom Tool Flow (Function Calling)

Custom tools (function calling) require your application's involvement:

  1. The model generates structured JSON requesting a function call (with name, args, and unique ID)
  2. Your application executes the function
  3. Your application returns results to the model
  4. The model generates its final response

Combining Built-in and Custom Tools Flow (Preview)

A preview feature allows combining both built-in and custom tools simultaneously using tool context circulation to coordinate execution across different environments.

Supported model endpoint: gemini-3.1-pro-preview-customtools

Required configuration when mixing Google Search with custom functions:

include_server_side_tool_invocations=True

Structured Outputs vs. Function Calling

Structured outputs addresses schema compliance for final responses — ensuring the model's answer conforms to a specified JSON schema.

Function calling handles intermediate steps requiring external system integration — the model calls your functions to gather data before forming a response.

Structured Outputs with Tools

Gemini 3 series models support combining structured outputs with built-in tools. This ensures responses maintain required schemas while being grounded in external data (e.g., Google Search results). The differentiation matters: function calling is for intermediate tool steps, while structured output governs the final response format.
