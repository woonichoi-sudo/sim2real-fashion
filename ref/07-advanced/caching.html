Context Caching in Google Gemini API

Overview

The Gemini API provides two caching mechanisms: implicit caching (automatic, no cost guarantee) and explicit caching (manual, with cost savings guarantee).

Implicit Caching

Status: Enabled by default for Gemini 2.5 and newer models. Requires no developer setup.

Minimum Token Limits by Model:

ModelMin Token Limit
Gemini 3.5 Flash1024
Gemini 3 Pro Preview4096
Gemini 2.5 Flash1024
Gemini 2.5 Pro4096

Optimization strategies: Place large common content early in prompts; send similar requests within short timeframes.

Explicit Caching

Key Configuration Parameters

  • TTL (Time to Live): Default is 1 hour; configurable in seconds
  • Display Name: Optional identifier for cache management (max 128 Unicode characters)
  • System Instruction: Can be cached along with content

Core Operations

Create Cache - Python

cache = client.caches.create(
    model=model,
    config=types.CreateCachedContentConfig(
        display_name='cache_identifier',
        system_instruction='Your instruction here',
        contents=[content],
        ttl="300s",
    )
)

Generate Using Cache - Python

response = client.models.generate_content(
    model=model,
    contents='Your query',
    config=types.GenerateContentConfig(cached_content=cache.name)
)

List Caches

Python:

for cache in client.caches.list():
    print(cache)
client.caches.get(name=name)

JavaScript:

const pager = await ai.caches.list({ config: { pageSize: 10 } });

REST: GET https://generativelanguage.googleapis.com/v1beta/cachedContents?key=$GEMINI_API_KEY

Update Cache

Python - TTL:

client.caches.update(
    name=cache.name,
    config=types.UpdateCachedContentConfig(ttl='300s')
)

Python - Expire Time:

import datetime
in10min = datetime.datetime.now(datetime.timezone.utc) + datetime.timedelta(minutes=10)
client.caches.update(
    name=cache.name,
    config=types.UpdateCachedContentConfig(expire_time=in10min)
)

JavaScript:

const ttl = `${2 * 3600}s`;
const updatedCache = await ai.caches.update({ name: cache.name, config: { ttl } });

REST:

curl -X PATCH "https://generativelanguage.googleapis.com/v1beta/$CACHE_NAME?key=$GEMINI_API_KEY" \
    -H 'Content-Type: application/json' \
    -d '{"ttl": "600s"}'

Delete Cache

Python: client.caches.delete(cache.name)

JavaScript: await ai.caches.delete({ name: cache.name });

REST: curl -X DELETE "https://generativelanguage.googleapis.com/v1beta/$CACHE_NAME?key=$GEMINI_API_KEY"

Video Processing Example

import requests, pathlib, time
from google import genai
from google.genai import types

client = genai.Client()
url = 'https://storage.googleapis.com/generativeai-downloads/data/SherlockJr._10min.mp4'
path = pathlib.Path('SherlockJr._10min.mp4')
path.write_bytes(requests.get(url).content)

video_file = client.files.upload(file=path)
while video_file.state.name == 'PROCESSING':
    time.sleep(2.5)
    video_file = client.files.get(name=video_file.name)

cache = client.caches.create(
    model='models/gemini-3.5-flash',
    config=types.CreateCachedContentConfig(
        display_name='sherlock jr movie',
        system_instruction='You are an expert video analyzer...',
        contents=[video_file],
        ttl="300s",
    )
)

response = client.models.generate_content(
    model='models/gemini-3.5-flash',
    contents='Introduce different characters...',
    config=types.GenerateContentConfig(cached_content=cache.name)
)

PDF Processing Example

from google import genai
from google.genai import types
import io, httpx

client = genai.Client()
doc_io = io.BytesIO(httpx.get("https://sma.nasa.gov/SignificantIncidents/assets/a11_missionreport.pdf").content)

document = client.files.upload(file=doc_io, config=dict(mime_type='application/pdf'))

cache = client.caches.create(
    model="gemini-3.5-flash",
    config=types.CreateCachedContentConfig(
        system_instruction="You are an expert analyzing transcripts.",
        contents=[document],
    )
)

response = client.models.generate_content(
    model="gemini-3.5-flash",
    contents="Please summarize this transcript",
    config=types.GenerateContentConfig(cached_content=cache.name)
)

JavaScript Example

import { GoogleGenAI, createUserContent, createPartFromUri } from "@google/genai";

const ai = new GoogleGenAI({ apiKey: "GEMINI_API_KEY" });

async function main() {
    const doc = await ai.files.upload({ file: "path/to/file.txt", config: { mimeType: "text/plain" } });

    const cache = await ai.caches.create({
        model: "gemini-3.5-flash",
        config: {
            contents: createUserContent(createPartFromUri(doc.uri, doc.mimeType)),
            systemInstruction: "You are an expert analyzing transcripts.",
        },
    });

    const response = await ai.models.generateContent({
        model: "gemini-3.5-flash",
        contents: "Please summarize this transcript",
        config: { cachedContent: cache.name },
    });
}

Go Example

cache, err := client.Caches.Create(ctx, modelName, &genai.CreateCachedContentConfig{
    Contents: contents,
    SystemInstruction: genai.NewContentFromText("You are an expert analyzing transcripts.", genai.RoleUser),
})

response, err := client.Models.GenerateContent(
    ctx,
    modelName,
    genai.Text("Please summarize this transcript"),
    &genai.GenerateContentConfig{ CachedContent: cache.Name },
)

REST API Workflow

Create Cache:

curl -X POST "https://generativelanguage.googleapis.com/v1beta/cachedContents?key=$GEMINI_API_KEY" \
    -H 'Content-Type: application/json' \
    -d @request.json

Generate Content:

curl -X POST "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent?key=$GEMINI_API_KEY" \
    -H 'Content-Type: application/json' \
    -d '{"contents": [...], "cachedContent": "'"$CACHE_NAME"'"}'

Use Cases

"Context caching is particularly well suited to scenarios where a substantial initial context is referenced repeatedly by shorter requests." Optimal for:

  • Chatbots with extensive system instructions
  • Repetitive video file analysis
  • Recurring queries against large document sets
  • Code repository analysis and bug fixing

Cost Structure

Billing based on:

  1. Cache token count: Input tokens cached at reduced rate
  2. Storage duration (TTL): Billed based on TTL duration
  3. Other factors: Non-cached input and output tokens

Reference: Gemini API Pricing Page for current rates.

Additional Constraints

  • Minimum token count varies by model; maximum equals model's standard limit
  • No distinction made between cached and regular input tokens
  • Standard GenerateContent rate limits apply
  • Cached token count returned in usage_metadata field
  • TTL format: duration in seconds with up to 9 fractional digits, ending with 's'
  • Cached content is model-specific: can only be used with the model it was created for
