Google Gemini API - Batch API Reference

Overview

Google's Batch API enables processing multiple requests in a single call through the Gemini API, supporting both content generation and embedding operations.

비용 및 처리 시간: Batch API는 요청을 비동기적으로 처리하며 동기(표준 interactive) API 대비 50% 비용 절감(표준 비용의 50% 가격)을 제공합니다. 목표 처리 시간은 24시간 이내이며, 대부분의 작업은 작업 규모와 시스템 부하에 따라 더 빠르게 완료됩니다.

Core Methods

1. models.batchGenerateContent

Endpoint: POST https://generativelanguage.googleapis.com/v1beta/{batch.model=models/*}:batchGenerateContent

Purpose: Enqueues batches of generateContent requests for asynchronous processing.

Path Parameter: batch.model (string, required) - Format: models/{model}

2. models.asyncBatchEmbedContent

Endpoint: POST https://generativelanguage.googleapis.com/v1beta/{batch.model=models/*}:asyncBatchEmbedContent

Purpose: Enqueues batches of embedContent requests for asynchronous processing. Distinguishes itself from the synchronous handler to avoid naming conflicts.

3. batches.get

Endpoint: GET https://generativelanguage.googleapis.com/v1beta/{name=batches/*}

Purpose: Retrieves the latest state of a long-running operation.

4. batches.list

Endpoint: GET https://generativelanguage.googleapis.com/v1beta/{name=batches}

Query Parameters: filter, pageSize, pageToken, returnPartialSuccess

5. batches.cancel

Endpoint: POST https://generativelanguage.googleapis.com/v1beta/{name=batches/*}:cancel

Purpose: Initiates asynchronous cancellation of operations.

6. batches.delete

Endpoint: DELETE https://generativelanguage.googleapis.com/v1beta/{name=batches/*}

7. batches.updateGenerateContentBatch

Endpoint: PATCH https://generativelanguage.googleapis.com/v1beta/{generateContentBatch.name=batches/*}:updateGenerateContentBatch

Query Parameter: updateMask (FieldMask format)

8. batches.updateEmbedContentBatch

Endpoint: PATCH https://generativelanguage.googleapis.com/v1beta/{embedContentBatch.name=batches/*}:updateEmbedContentBatch

Key Data Structures

GenerateContentBatch

Fields:

  • model (string, required) - Format: models/{model}
  • name (string, output-only) - Format: batches/{batchId}
  • displayName (string, required)
  • inputConfig (InputConfig object, required)
  • output (GenerateContentBatchOutput, output-only)
  • createTime, endTime, updateTime (Timestamp)
  • batchStats (BatchStats)
  • state (BatchState enum)
  • priority (int64, optional) - Higher values processed first; negative values permitted, defaulting to zero

InputConfig

Union type with sources:

  • fileName (string) - Reference to File containing requests
  • requests (InlinedRequests) - Embedded request objects

GenerateContentRequest

Key fields:

  • model (string, required)
  • contents[] (Content objects) - Conversation history/current query
  • tools[] (Tool objects) - Function calling or code execution
  • toolConfig (ToolConfig)
  • safetySettings[] (SafetySetting objects)
  • systemInstruction (Content, text-only)
  • generationConfig (GenerationConfig)
  • cachedContent (string) - Format: cachedContents/{cachedContent}
  • serviceTier (ServiceTier enum)
  • store (boolean)

EmbedContentRequest

Fields:

  • model (string, required) - Format: models/{model}
  • content (Content, required) - Only parts.text counted
  • taskType (TaskType enum, deprecated)
  • title (string, deprecated)
  • outputDimensionality (integer, deprecated)
  • embedContentConfig (EmbedContentConfig)

BatchStats

  • requestCount (int64)
  • successfulRequestCount (int64)
  • failedRequestCount (int64)
  • pendingRequestCount (int64)

Operation

  • name (string) - Unique identifier within service
  • metadata (object) - Service-specific progress information
  • done (boolean) - Completion status
  • error (Status) - Error result
  • response (object) - Success result

BatchState Enum

  • BATCH_STATE_UNSPECIFIED
  • BATCH_STATE_PENDING
  • BATCH_STATE_RUNNING
  • BATCH_STATE_SUCCEEDED
  • BATCH_STATE_FAILED
  • BATCH_STATE_CANCELLED
  • BATCH_STATE_EXPIRED

ServiceTier Enum

  • unspecified (default/standard)
  • standard
  • flex
  • priority

Output Formats

GenerateContentBatchOutput

Union type:

  • responsesFile (string) - JSONL file with GenerateContentResponse objects (one response per line)
  • inlinedResponses (InlinedResponses) - Embedded response objects

InlinedResponse

  • metadata (Struct, output-only)
  • error (Status) - Processing error
  • response (GenerateContentResponse) - Actual response

Request/Response Flow

  1. Submit batch via batchGenerateContent or asyncBatchEmbedContent
  2. Returns Operation object with long-running operation ID
  3. Poll using batches.get to monitor progress
  4. Retrieve results from output field when done=true
  5. Optional: Cancel with batches.cancel or delete with batches.delete

Key Features Summary

  • Priority System: "Batches with a higher priority value will be processed before batches with a lower priority value." Negative values are permitted, defaulting to zero.
  • Flexible Input: Batches accept either inlined requests within the API call or references to uploaded files containing requests.
  • Output Options: Results can be returned as inlined responses or stored in JSONL files with one response per line.
  • Status Tracking: BatchStats provides counts for total, successful, failed, and pending requests.
  • Batch States: Operations progress through PENDING → RUNNING → SUCCEEDED/FAILED/CANCELLED/EXPIRED states.
