Google Gemini API - Caching API Reference

Overview

The Gemini API's caching feature allows developers to save and reuse preprocessed input tokens for repeated queries, enabling cost and speed savings when asking multiple questions about the same content.

API Methods

1. cachedContents.create

Endpoint: POST https://generativelanguage.googleapis.com/v1beta/cachedContents

Purpose: Creates a CachedContent resource by preprocessing content (files, text, tools) for reuse across requests.

Key Request Fields:

  • contents[] (Content objects) - immutable content to cache
  • tools[] (Tool objects) - optional tools the model may use
  • model (string, required) - format: models/{model}
  • systemInstruction (Content) - developer system instructions
  • displayName (string) - max 128 Unicode characters
  • expiration (union type):
    • expireTime (RFC 3339 timestamp)
    • ttl (duration format, e.g., "300s")
  • toolConfig (ToolConfig) - shared tool configuration

2. cachedContents.list

Endpoint: GET https://generativelanguage.googleapis.com/v1beta/cachedContents

Purpose: Retrieves paginated list of cached content entries with optional filtering parameters.

Query Parameters:

  • pageSize (integer) - max 1000 items
  • pageToken (string) - pagination token

Response: List of CachedContent objects with nextPageToken

3. cachedContents.get

Endpoint: GET https://generativelanguage.googleapis.com/v1beta/{name=cachedContents/*}

Purpose: Fetches a specific cached content resource by its identifier.

Path Parameter: name (string) - format: cachedContents/{id}

Returns: Single CachedContent instance

4. cachedContents.patch

Endpoint: PATCH https://generativelanguage.googleapis.com/v1beta/{cachedContent.name=cachedContents/*}

Purpose: Updates only the expiration settings (TTL or expireTime) of existing cached content.

Query Parameter: updateMask (FieldMask) - fields to update (comma-separated)

Updatable Fields: Only expiration (expireTime or ttl)

Note: "only expiration is updatable"

5. cachedContents.delete

Endpoint: DELETE https://generativelanguage.googleapis.com/v1beta/{name=cachedContents/*}

Purpose: Removes a cached content resource entirely.

Returns: Empty JSON object {} on success

Code Examples by SDK

Python

from google import genai
from google.genai import types

client = genai.Client()
document = client.files.upload(file=media / "a11.txt")
model_name = "gemini-3.5-flash"

cache = client.caches.create(
    model=model_name,
    config=types.CreateCachedContentConfig(
        contents=[document],
        system_instruction="You are an expert analyzing transcripts.",
    ),
)

response = client.models.generate_content(
    model=model_name,
    contents="Please summarize this transcript",
    config=types.GenerateContentConfig(cached_content=cache.name),
)

Update TTL:

import datetime
ttl = f"{int(datetime.timedelta(hours=2).total_seconds())}s"
client.caches.update(
    name=cache.name, config=types.UpdateCachedContentConfig(ttl=ttl)
)

Get Cache:

cache = client.caches.get(name=cache_name)

Delete Cache:

client.caches.delete(name=cache.name)

Node.js

const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });
const document = await ai.files.upload({
  file: filePath,
  config: { mimeType: "text/plain" },
});

const cache = await ai.caches.create({
  model: modelName,
  config: {
    contents: contents,
    systemInstruction: "You are an expert analyzing transcripts.",
  },
});

const response = await ai.models.generateContent({
  model: modelName,
  contents: "Please summarize this transcript",
  config: { cachedContent: cache.name },
});

Update TTL:

const ttl = `${2 * 3600}s`;
cache = await ai.caches.update({
  name: cache.name,
  config: { ttl },
});

Go

ctx := context.Background()
client, err := genai.NewClient(ctx, &genai.ClientConfig{
    APIKey:  os.Getenv("GEMINI_API_KEY"),
    Backend: genai.BackendGeminiAPI,
})

cache, err := client.Caches.Create(ctx, modelName, &genai.CreateCachedContentConfig{
    Contents: contents,
    SystemInstruction: genai.NewContentFromText(
        "You are an expert analyzing transcripts.", genai.RoleUser,
    ),
})

response, err := client.Models.GenerateContent(
    ctx,
    modelName,
    genai.Text("Please summarize this transcript"),
    &genai.GenerateContentConfig{
        CachedContent: cache.Name,
    },
)

Shell/cURL

curl -X POST "https://generativelanguage.googleapis.com/v1beta/cachedContents?key=$GEMINI_API_KEY" \
 -H 'Content-Type: application/json' \
 -d @request.json

curl "https://generativelanguage.googleapis.com/v1beta/$CACHE_NAME?key=$GEMINI_API_KEY"

curl -X PATCH "https://generativelanguage.googleapis.com/v1beta/$CACHE_NAME?key=$GEMINI_API_KEY" \
 -H 'Content-Type: application/json' \
 -d '{"ttl": "600s"}'

curl -X DELETE "https://generativelanguage.googleapis.com/v1beta/$CACHE_NAME?key=$GEMINI_API_KEY"

Advanced Use Case: Cache From Chat History

chat = client.chats.create(
    model=model_name,
    config=types.GenerateContentConfig(system_instruction=system_instruction),
)
document = client.files.upload(file=media / "a11.txt")

response = chat.send_message(["Hi, could you summarize this transcript?", document])
response = chat.send_message(["Okay, could you tell me more about the trans-lunar injection"])

cache = client.caches.create(
    model=model_name,
    config={
        "contents": chat.get_history(),
        "system_instruction": system_instruction,
    },
)

chat = client.chats.create(
    model=model_name,
    config=types.GenerateContentConfig(cached_content=cache.name),
)
response = chat.send_message("I didn't understand that last part, could you explain it simpler?")

Data Models

CachedContent Resource

Output Fields:

  • name (string) - resource identifier
  • createTime (RFC 3339 timestamp)
  • updateTime (RFC 3339 timestamp)
  • usageMetadata - token consumption data
  • expireTime (always provided on output)

Content Structure

{
  "parts": [Part objects],
  "role": "user" | "model"
}

Part Types (Union)

  • text (string)
  • inlineData (Blob with mimeType + base64 data)
  • fileData (FileData with fileUri)
  • functionCall, functionResponse
  • executableCode, codeExecutionResult
  • toolCall, toolResponse
  • videoMetadata (deprecated)

Tool Configuration

{
  "functionDeclarations[]": FunctionDeclaration[],
  "googleSearchRetrieval": GoogleSearchRetrieval,
  "codeExecution": CodeExecution,
  "googleSearch": GoogleSearch,
  "computerUse": ComputerUse,
  "urlContext": UrlContext,
  "fileSearch": FileSearch,
  "mcpServers[]": McpServer[],
  "googleMaps": GoogleMaps
}

FunctionDeclaration

  • name (string, required) - a-z, A-Z, 0-9, underscores, colons, dots, dashes; max 128 chars
  • description (string, required)
  • parameters (Schema) or parametersJsonSchema
  • response (Schema) or responseJsonSchema
  • behavior - BLOCKING | NON_BLOCKING

Schema Types

Supports: STRING, NUMBER, INTEGER, BOOLEAN, ARRAY, OBJECT, NULL

Properties:

  • type, format, title, description
  • nullable, enum[], pattern
  • minLength, maxLength, minimum, maximum
  • minItems, maxItems, properties, required
  • anyOf[], items, example, default

Important Constraints

  • Cached content is model-specific: "can be only used with model it was created for"
  • Maximum display name: 128 Unicode characters
  • TTL format: duration in seconds with up to 9 fractional digits, ending with 's'
  • RFC 3339 timestamps use Z-normalization with 0, 3, 6, or 9 fractional digits
  • Function names limited to 128 characters
  • Schema type is required
  • updateMask required for PATCH operations (comma-separated field names)
  • Only expiration fields are patchable

Response Format

All successful responses return JSON objects. List operations include nextPageToken for pagination. Delete operations return empty JSON {}.

Language Support

Official SDKs support Python, Go, TypeScript, Java, and C#, with shell/curl examples provided for direct API access.
