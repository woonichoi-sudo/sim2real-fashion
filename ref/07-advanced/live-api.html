Live API - WebSockets API Reference

Overview

The Live API is a preview WebSocket-based service that enables stateful, bidirectional communication with Gemini. It allows clients to send and receive text, audio, and video in real-time conversations.

WebSocket Endpoint

wss://generativelanguage.googleapis.com/ws/google.ai.generativelanguage.v1beta.GenerativeService.BidiGenerateContent

Session Configuration

Initial setup messages configure the session and include:

  • Model
  • Generation parameters
  • System instructions
  • Tools

Important: "You cannot update the configuration while the connection is open."

BidiGenerateContentSetup.generationConfig

The generationConfig object inside BidiGenerateContentSetup controls generation behavior. Its structure (per the official Live API reference):

"generationConfig": {
  "candidateCount": integer,
  "maxOutputTokens": integer,
  "temperature": number,
  "topP": number,
  "topK": integer,
  "presencePenalty": number,
  "frequencyPenalty": number,
  "responseModalities": [string],
  "speechConfig": object,
  "mediaResolution": object
}
FieldTypeDescription
responseModalities [string] (array of Modality enum) The requested modalities of the response. Represents the set of modalities the model can return; must be a subset of the model's supported modalities. Allowed values: TEXT, AUDIO, IMAGE (Modality enum). The Live API reference links this field to GenerationConfig; the enumerated Modality values are defined in the GenerationConfig reference (개별 enum 값은 GenerationConfig 참조 문서 기준, Live API 페이지 자체에는 미열거).

Client Message Format

JSON objects must contain exactly one field:

{
  "setup": BidiGenerateContentSetup,
  "clientContent": BidiGenerateContentClientContent,
  "realtimeInput": BidiGenerateContentRealtimeInput,
  "toolResponse": BidiGenerateContentToolResponse
}

Client Messages

MessageDescription
BidiGenerateContentSetupFirst message sent; initial configuration for the session
BidiGenerateContentClientContentIncremental content updates from the client / conversation updates
BidiGenerateContentRealtimeInputReal-time audio/video/text input
BidiGenerateContentToolResponseResponses to tool calls from the server

Server Messages

BidiGenerateContentServerMessage: Response message containing one of:

  • setupComplete - Confirms configuration
  • serverContent - Model-generated responses
  • toolCall - Function execution requests
  • toolCallCancellation
  • goAway - Disconnection notice
  • sessionResumptionUpdate

Receiving Server Messages - Python Example

async with client.aio.live.connect(model='...', config=config) as session:
    await session.send(input='Hello world!', end_of_turn=True)
    async for message in session.receive():
        print(message)

GenerationConfig Fields Not Supported in Live API

  • responseLogprobs
  • responseMimeType
  • logprobs
  • responseSchema
  • stopSequence
  • routingConfig
  • audioTimestamp

Real-time Input Features

The API supports automatic activity detection with configurable sensitivity for:

  • Speech start/end detection
  • Silence duration
  • Interruption handling

Clients can enable transcription for both input and output audio.

Session Management

Session resumption tokens allow reconnecting to previous conversations via SessionResumptionConfig. The server sends SessionResumptionUpdate messages. The system provides compression mechanisms to manage context window size automatically when configured.

Authentication - Ephemeral Tokens

Ephemeral tokens can be created via AuthTokenService.CreateToken (AuthToken) with configurable expiration times:

  • Default: 30 minutes
  • Maximum: 20 hours

Tokens can be passed as query parameters or in the "Authorization" header.
