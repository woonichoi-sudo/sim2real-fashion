Interactions API Overview

The Interactions API is Google's newer approach to accessing Gemini models, positioned as an upgrade from the classic generateContent API. It remains in beta stage.

Core Benefits

  • Server-side history management: previous_interaction_id를 통해 단순화된 다중 턴 흐름 구현. Developers can reference previous conversations rather than resending full chat histories.
  • Observable execution steps: 디버깅과 UI 렌더링을 위한 타입화된 단계별 추적
  • Built for agentic workflows: Native support for multi-step tool use, orchestration, and complex reasoning flows.
  • Long-running and background tasks: background=true로 Deep Think/Deep Research 작업 백그라운드 처리

Core Resource: Interaction Object

An Interaction represents "a complete turn in a conversation or task" and contains a time-ordered sequence of execution steps.

SDK Convenience Properties

PropertyReturn TypeDescription
interaction.output_textStringReturns the last text block
interaction.output_imageImageContentReturns the last generated image
interaction.output_audioAudioContentReturns the last generated audio

SDK Support

  • Python: google-genai v1.55.0+
  • JavaScript: @google/genai v1.33.0+

Supported Models

Gemini 3.5 Flash, 3.1 Pro Preview, 2.5 Pro/Flash, Deep Research agents, and others.

Data Retention

By default, interactions are stored with retention periods based on tier:

  • Paid users: 55 days
  • Free tier: 1 day

Users can opt out via store=false, though this prevents background execution and server-side state features.

Current Status and Limitations

The API remains in beta, with breaking changes possible. Features "not yet available" in Interactions API (available in generateContent):

  • Video metadata
  • Batch API
  • Explicit caching (implicit caching is supported)

For production workloads, the established generateContent API remains recommended.

The page emphasizes this is suited for new projects and agentic workflows, while existing stable integrations can continue using the original API.

streamGenerateContent (스트리밍 생성)

streamGenerateContent는 generateContent의 스트리밍 변형으로, 응답이 완전히 생성될 때까지 기다리지 않고 생성되는 대로 증분(incremental) 청크를 받아 실시간 출력을 구현합니다.

Endpoint

HTTP 메서드와 경로:

POST https://generativelanguage.googleapis.com/v1beta/{model=models/*}:streamGenerateContent

마스터 문서 §13 기준 경로: POST /v1beta/models/{model}:streamGenerateContent. SSE 형식으로 받으려면 쿼리 파라미터 alt=sse를 추가합니다 (예: ...:streamGenerateContent?alt=sse).

Request Body

요청 본문 구조는 generateContent와 동일합니다. 공식 문서 기준 동일 필드:

  • contents[]
  • tools[]
  • toolConfig
  • safetySettings[]
  • systemInstruction
  • generationConfig
  • cachedContent
  • serviceTier
  • store

Response Format (SSE)

성공 시 응답 본문은 Server-Sent Events(SSE) 스트림으로, GenerateContentResponse 인스턴스가 생성되는 순서대로 증분 전달됩니다. 공식 문서 원문: "If successful, the response body contains a stream of GenerateContentResponse instances."

REST 예시 (curl)

curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:streamGenerateContent?alt=sse&key=${GEMINI_API_KEY}" \
  --no-buffer \
  -H 'Content-Type: application/json' \
  -d '{ "contents": [{ "parts": [{ "text": "Explain how AI works" }] }] }'

--no-buffer 옵션과 alt=sse 파라미터로 SSE 스트리밍을 활성화합니다.

SDK 예시 (Python)

response = client.models.generate_content_stream(
    model="gemini-3.5-flash",
    contents=["Explain how AI works"]
)
for chunk in response:
    print(chunk.text, end="")

SDK에서는 응답 청크를 순회(iterate)하며 도착하는 대로 처리합니다 (전체 생성 완료를 기다리지 않음).
