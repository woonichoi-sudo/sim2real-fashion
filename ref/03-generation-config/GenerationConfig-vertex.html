GenerationConfig - Overview

Configuration options for model generation and outputs. Not all parameters are configurable for every model.

GenerationConfig Fields

ParameterTypeDescription
stopSequences[]stringThe set of character sequences (up to 5) that will stop output generation. If specified, the API will stop at the first appearance of a stop sequence. The stop sequence will not be included as part of the response.
responseMimeTypestringOutput response mimetype of the generated candidate text. Supported mimetype: text/plain (default), application/json, text/x.enum
responseSchemaSchema objectOutput response schema of the generated candidate text when response MIME type can have schema. Schema can be objects, primitives or arrays and is a subset of OpenAPI schema.
responseModalities[]string (enum Modality)The requested modalities of the response. Represents the set of modalities that the model can return. Defaults to MODALITY_UNSPECIFIED which will default to the modalities supported by the request.
candidateCountintegerNumber of generated responses to return. Currently, this value can only be set to 1. If unset, this will default to 1.
maxOutputTokensintegerThe maximum number of tokens to include in a response candidate. Note: The default value varies by model.
temperaturenumberControls the randomness of the output. Note: The default value varies by model. Values can range from [0.0, 2.0].
topPnumberThe maximum cumulative probability of tokens to consider when sampling. The model uses combined Top-k and Top-p (nucleus) sampling. Tokens are sorted based on their assigned probabilities so that only the most likely tokens are considered. Top-k sampling directly limits the maximum number of tokens to consider, while Nucleus sampling limits the number of tokens based on the cumulative probability.
topKintegerThe maximum number of tokens to consider when sampling. Gemini models use Top-p (nucleus) sampling or a combination of Top-k and nucleus sampling. Top-k sampling considers the set of topK most probable tokens. Models running with nucleus sampling don't allow topK setting.
seedintegerSeed used in decoding. If not set, the request uses a randomly generated seed.
presencePenaltynumberPresence penalty applied to the next token's logprobs if the token has already been seen in the response. This penalty is binary on/off and not dependent on the number of times the token is used (after the first). Use frequencyPenalty for a penalty that increases with each use.
frequencyPenaltynumberFrequency penalty applied to the next token's logprobs, multiplied by the number of times each token has been seen in the response so far.
logprobsintegerOnly valid if responseLogprobs=True. This sets the number of top logprobs to return at each decoding step in the Candidate.logprobs_result.
responseLogprobsbooleanIf true, export the logprobs results in response.
enableEnhancedCivicAnswersbooleanOptional. Enables enhanced civic answers. It may not be available for all models.
speechConfigSpeechConfig objectThe speech generation config.
thinkingConfigThinkingConfig objectOptional. Config for thinking features.
mediaResolutionenum (MediaResolution)Optional. If specified, the media resolution specified will be used.
audioTimestampbooleanOptional. If enabled, audio timestamp will be included in the request to the model.
routingConfigGenerationConfig.RoutingConfig objectOptional. Routing configuration.
