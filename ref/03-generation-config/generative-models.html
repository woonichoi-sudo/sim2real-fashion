Model Parameters

  • Max output tokens: "Specifies the maximum number of tokens that can be generated in the response."
  • Temperature: Controls randomness in token selection. Range: 0 (deterministic) to higher values (creative). Note for Gemini 3 models: "we strongly recommend keeping the temperature at its default value of 1.0."
  • topK: Selects next token from highest-probability options. Value of 1 means greedy decoding.
  • topP: Tokens selected until cumulative probability equals this value. Default: 0.95
  • stop_sequences: "Set a stop sequence to tell the model to stop generating content."

Zero-shot vs Few-shot

"Prompts without few-shot examples are likely to be less effective. In fact, you can remove instructions from your prompt if your examples are clear enough in showing the task at hand."

Temperature Note

"When using Gemini 3 models, we strongly recommend keeping the temperature at its default value of 1.0. Changing the temperature (setting it below 1.0) may lead to unexpected behavior."

Gemini 3 Core Principles

  • "Be precise and direct: State your goal clearly and concisely."
  • Use consistent structure with XML tags or Markdown headings
  • Define parameters explicitly
  • Control output verbosity
  • Handle multimodal inputs coherently
  • Prioritize critical instructions in system instruction or prompt beginning
  • "Structure for long contexts: When providing large amounts of context...supply all the context first."
  • "Anchor context: After a large block of data, use a clear transition phrase."
