Gemini Image Generation

Google's Gemini API supports native image generation through three models (collectively called "Nano Banana" in internal branding). All generated images include a SynthID watermark for responsible AI identification.

Supported Models

Model NameModel CodeDescription
Nano Banana 2 (Gemini 3.1 Flash Image)gemini-3.1-flash-imageSpeed and high-volume developer use cases (GA)
Nano Banana Pro (Gemini 3 Pro Image)gemini-3-pro-imageProfessional asset production, advanced reasoning (GA)
Nano Banana (Gemini 2.5 Flash Image)gemini-2.5-flash-imageSpeed and efficiency (GA)

Core Capabilities

1. Image Generation (Text-to-Image)

Create images from text prompts alone. Supports generating complete UI apps from prompts.

Python Example

from google import genai
from google.genai import types

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3.1-flash-image",
    contents="Generate an image of a cat sitting on a beach at sunset.",
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"]
    )
)

for part in response.candidates[0].content.parts:
    if part.inline_data:
        # Save or display the image
        image_data = part.inline_data.data

2. Image Editing (Text-and-Image-to-Image)

"Add, remove, or modify elements, change the style, or adjust the color grading" using text and existing images.

Python Example

import PIL.Image

image = PIL.Image.open("input.jpg")
response = client.models.generate_content(
    model="gemini-3.1-flash-image",
    contents=["Add a rainbow to this image", image],
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"]
    )
)

Iterative image refinement through multi-turn chat conversations.

5. High Resolution Output

Generation at 512 (0.5K), 1K, 2K, and 4K resolutions.

Note: Use uppercase "K" — 4K not 4k.

6. Grounding with Google Search

Real-time data integration for weather, events, and current information.

7. Grounding with Google Search for Images (Gemini 3.1 Flash only)

Visual context from web images. Requires source attribution and webpage links.

8. Video-to-Image Generation (Gemini 3.1 Flash only)

"Generate new images using a video's context as a multimodal reference." Pass a video file (via direct upload or public YouTube URL) alongside a text prompt to generate thumbnails, movie posters, etc.

Thinking Mode

Models use reasoning processes for complex prompts with adjustable thinking levels. Thinking tokens are billed regardless of whether include_thoughts is set.

Controlling Thinking Levels

config=types.GenerateContentConfig(
    thinking_config=types.ThinkingConfig(
        thinking_level="High"  # or "minimal"
    ),
    response_modalities=["TEXT", "IMAGE"]
)

Thought Signatures

Official SDKs automatically handle Thought Signatures for maintaining context across multi-turn thinking conversations.

Text Rendering

"Capable of generating legible, stylized text for infographics, menus, diagrams." Best results with concise text.

REST (curl) Example

curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-image:generateContent?key=$GEMINI_API_KEY" \
  -H 'Content-Type: application/json' \
  -d '{
    "contents": [{
      "parts": [{"text": "Generate an image of a cat sitting on a beach"}]
    }],
    "generationConfig": {
      "responseModalities": ["TEXT", "IMAGE"]
    }
  }'

Important Notes

  • All generated images include a SynthID watermark.
  • Ensure you have necessary rights for any uploaded images.
  • Use uppercase "K" for resolution sizes (e.g., 4K not 4k).
  • Thinking tokens are billed regardless of include_thoughts setting.
  • When using Image Search grounding, source attribution and webpage links are required.
  • Video-to-image is available only on Gemini 3.1 Flash (gemini-3.1-flash-image).
